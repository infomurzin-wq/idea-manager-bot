# Idea Manager Bot

Telegram-бот для захвата, маршрутизации и развития идей и контекста поверх текущего workspace `MYCODEX`.

## Что умеет MVP

- принимать идеи текстом, ссылкой, голосовым сообщением и аудио;
- принимать контекст текстом, ссылкой, голосовым сообщением и аудио;
- уметь обрабатывать ссылки как отдельный сценарий входа;
- автоматически определять проект:
  - `ufc-betting`
  - `venture-investing`
  - `learning-programming`
  - `bank-factoring-product`
  - `shared`
- сохранять карточку идеи в двух видах:
  - JSON как системную запись;
  - Markdown в соответствующую папку проекта;
- сохранять контекст как отдельную запись без обсуждения и анализа;
- при ссылке сохранять и сам URL, и извлечённое содержимое страницы;
- давать быстрый ответ по идее:
  - краткое усиление;
  - аргументы `за`;
  - аргументы `против`;
  - следующие шаги;
- показывать списки идей и контекста по проекту;
- открывать записи кнопками без ручного ввода ID;
- добавлять комментарии к идее с сохранением преемственности.

## Почему такая архитектура

У тебя уже есть файловая база знаний. Поэтому бот не строится вокруг отдельной тяжелой БД.

Источник истины здесь такой:

1. JSON хранит удобную для бота структуру.
2. Markdown-файл хранит человекочитаемую карточку идеи в нужном разделе workspace.

Так мы не теряем совместимость с текущей системой заметок.

## Интерфейс бота

Основной сценарий теперь кнопочный:

1. Открываешь бота.
2. Нажимаешь `Новая идея` или `Новый контекст`.
3. Выбираешь нужный раздел.
4. Отправляешь текст, ссылку, голосовое или аудио.
5. Бот сохраняет материал в соответствующую папку.

Для идеи:

- бот вернёт усиление, плюсы, сомнения и следующий шаг;
- после сохранения покажет кнопки `Открыть идею`, `Добавить комментарий`, `Назад к списку`.

Для контекста:

- бот ничего не развивает и не обсуждает;
- просто сохраняет материал в выделенную папку контекста;
- после сохранения показывает кнопки:
  - `Открыть контекст`
  - `Попросить summary`
  - `Сделать из контекста идею`
  - `Назад к списку`

## Что можно делать в списках

### Список идей

После открытия списка идей пользователь может:

- открыть конкретную идею;
- прочитать исходный текст, анализ и историю комментариев;
- нажать `Добавить комментарий`;
- вернуться к списку идей проекта.

### Список контекста

После открытия списка контекста пользователь может:

- открыть конкретный контекст;
- прочитать сохранённый материал;
- увидеть исходную ссылку, если контекст был отправлен ссылкой;
- увидеть извлечённое содержимое страницы, если бот смог его прочитать;
- вернуться к списку контекста проекта.

Контекст по умолчанию не обсуждается и не развивается, но служит накопительным слоем знаний для проекта.

По кнопке `Попросить summary` бот делает краткую выжимку материала.

По кнопке `Сделать из контекста идею` бот создаёт новую карточку идеи в том же проекте и сразу строит по ней анализ.

## Как бот работает со ссылками

Если пользователь отправляет ссылку, бот должен пройти такой конвейер:

1. Сохранить исходный URL.
2. Попытаться открыть страницу и извлечь текст.
3. Сохранить извлечённое содержимое отдельно от исходного URL.
4. Дальше использовать это содержимое по типу записи.

### Если ссылка отправлена как идея

Бот должен:

- сохранить ссылку;
- сохранить извлечённое содержание страницы;
- прочитать материал;
- дать комментарий и усиление идеи уже не только по URL, а по содержимому страницы.

### Если ссылка отправлена как контекст

Бот должен:

- сохранить ссылку;
- сохранить извлечённое содержание страницы;
- не анализировать и не обсуждать материал;
- просто положить его в контекст проекта как полезный источник.

## Что делать, если ссылка не прочиталась

Если бот не смог получить содержание страницы, он не должен молча делать вид, что всё получилось.

Правильное поведение такое:

1. Сохранить сам URL как часть записи.
2. Явно сообщить, что содержимое страницы извлечь не удалось.
3. Предложить прислать текст вручную.

Пример ответа:

```text
Ссылку я сохранил, но содержимое страницы прочитать не смог.
Пришли, пожалуйста, текст статьи или ключевой фрагмент вручную, и я добавлю его в запись.
```

Причины такой ошибки могут быть разными:

- сайт блокирует автоматическое чтение;
- страница требует авторизацию;
- страница пустая для бота из-за JavaScript-рендеринга;
- временная сетевая ошибка.

## Команды

- `/start` — открыть стартовый экран
- `/menu` — открыть главное меню
- `/help` — подсказка по сценариям
- `/projects` — доступные проекты
- `/list ufc-betting` — резервный способ показать идеи проекта
- `/show IDEA_ID` — резервный способ открыть идею
- `/comment IDEA_ID текст` — резервный способ добавить комментарий

## Как бот определяет, куда сохранять материал

В новом UX пользователь сначала сам выбирает раздел кнопкой. Это надёжнее, чем автоматическая догадка.

За счёт этого:

1. идея сразу ложится в правильный проект;
2. контекст не требует дополнительной классификации;
3. снижается риск неправильной маршрутизации.

## Синхронизация для режима 24/7

Если бот живёт в облаке, он не может напрямую писать в локальную папку `MYCODEX` на Mac.

Поэтому для production-сценария используется такой поток:

1. Бот принимает запись.
2. Бот экспортирует JSON в `sync inbox`.
3. На Mac launcher делает `pull`.
4. Затем локальный `sync_inbox.py` раскладывает записи по папкам `MYCODEX`.
